# Review: Task 010

## Round 1

- [process-revision-complete] `_get_int` (agent.py:301-314) only handles dict-style usage access via `.get(key)`. Spec Section 3 (agent-architecture.md) explicitly requires: "Usage can be dict or object: Code handles both dict-style and attribute-style access patterns." If the SDK returns usage as an object with attributes, `_get_int` raises `AttributeError`. Both `accumulate_message` and `apply_final` are affected since they both call `_get_int`. The type annotation on `accumulate_message` (`dict[str, Any] | None`) also excludes object-style usage.
- [process-revision-complete] Stop hook (hooks.py:108-117) only logs `stop_hook_active`. Spec Section 6 (agent-architecture.md) table says the Stop hook purpose is "Log agent completion, total usage, cost." The `create_hooks` factory doesn't receive the `UsageStats` instance, so the Stop hook cannot access usage or cost data. Usage/cost is instead logged with a different event name (`extraction.agent_completed`) in the message loop (agent.py:217-221), but the spec assigns this responsibility to the Stop hook emitting `extraction.agent_stopped`.
- [process-revision-complete] PostToolUse hook (hooks.py:84-90) does not log the tool result. Spec Section 6 (agent-architecture.md) says PostToolUse should "Log tool result, duration, success/failure." The `tool_response` is read from `input_data` at line 79 only to check `is_error`; the actual result content is discarded. The hook logs tool name, duration, and is_error, but omits the result payload.
