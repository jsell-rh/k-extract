# Review: Task 008

## Round 1

- [process-revision-complete] `build_guidance_prompt` omits `optional_properties` for relationship types (`src/k_extract/extraction/prompts.py:66-73`). Entity types include both `required_properties` and `optional_properties` (lines 47-52), but relationship types only include `required_properties` — `optional_properties` is silently dropped despite being available on `RelationshipTypeConfig` (`src/k_extract/config/schema.py:60`). Per `specs/agent/prompt-generation.md` Layer 2: "Relationship type extraction guidance: what connections to look for, directionality, properties" — the LLM generating guidance will not know about optional relationship properties, producing incomplete guidance.

- [process-revision-complete] `substitute_job_variables` uses `str.format()` which crashes on user-edited templates containing literal braces (`src/k_extract/extraction/prompts.py:170`). `str.format()` interprets all `{...}` patterns as format fields. Per `specs/agent/prompt-generation.md`: "YAML is human-readable. Users can review and edit prompts before running." If a user edits `job_description_template` in the config and includes literal `{` or `}` characters (e.g., JSON notation, `{example}`), the substitution raises `KeyError`/`ValueError` at runtime. The spec says "Pure string substitution" — `str.format()` does more than substitution by parsing all brace patterns, not just the four documented placeholders.
